Exports are admin-only and contain anonymous analytics data. No child personal data is included.

What we collect

  • An anonymous browser ID (random UUID, stored in localStorage as gulsabi_anonymous_user_id)
  • Session and game-session UUIDs (used to stitch events together)
  • Device type, browser, OS (inferred from User-Agent)
  • UTM parameters from the URL when a campaign link is used
  • Game events: page views, game starts/completions/exits, score updates, retries, install lifecycle, errors

What we never collect

  • Names, emails, phone numbers, postal addresses
  • School names or any institution identifying details
  • Photos, voice, chat text, free-text user input
  • Exact GPS coordinates — country/region only, inferred coarsely
  • Any child personal data of any kind

How it stays safe

  • The frontend uses only the public Supabase ANON key. No service-role key is ever shipped to the browser.
  • Row Level Security policies allow public visitors to insert events but never read them back.
  • The admin dashboard reads analytics only through an authenticated Supabase session.
  • A defensive PII guard strips any obviously personal field names from event metadata before insert.
